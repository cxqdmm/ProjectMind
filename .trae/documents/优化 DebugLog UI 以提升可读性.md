为了提升 `DebugLog.vue` 的可读性，我将通过以下方式进行 UI 改造：

1.  **引入类型映射与图标**：
    *   根据日志标题（`log.title`）进行关键词匹配，自动分配不同的**图标**和**主题色**。
    *   例如：
        *   **模型/思考 (LLM)**: 🧠 (紫色/蓝色) - 对应 `模型调用`
        *   **任务/规划 (Task)**: 📋 (橙色) - 对应 `任务`, `规划`
        *   **工具 (Tool)**: 🛠️ (绿色) - 对应 `工具`
        *   **记忆 (Memory)**: 🧠 (青色) - 对应 `记忆`
        *   **决策 (Decision)**: ⚖️ (红色/品红) - 对应 `决策`, `Gate`

2.  **视觉分层设计**：
    *   **左侧标识条**：在每条日志左侧添加一条彩色竖线，快速区分类型。
    *   **标签化 (Badges)**：在标题旁添加小标签，突出核心动作（如 "Start", "Done", "Error"）。
    *   **结构化内容展示**：不再简单 `JSON.stringify`，而是针对特定类型的内容（如 `content.messages` 或 `content.tool`）做简单的格式化或高亮，或者保留 `pre` 但优化字体和背景。

3.  **具体实现步骤**：
    *   在 `DebugLog.vue` 中增加 `getLogType(title)` 函数，返回类型对象 `{ icon, color, label }`。
    *   修改 `template`，使用动态 class 和 style 应用主题色。
    *   优化 CSS，增加圆角、间距和更柔和的背景色，减少视觉疲劳。

通过这些改动，用户一眼就能通过颜色和图标识别出当前日志属于哪个处理阶段，从而极大提升调试效率。
